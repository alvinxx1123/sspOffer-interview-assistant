# Evaluation Rubric

维度说明：

- correctness：是否抓住核心结论，是否存在明显事实错误
- depth：是否解释到底层原理、实现细节、边界场景
- structure：回答是否清楚分层，有主线和逻辑顺序
- communication：表达是否清晰、简洁、可理解
- risk：是否提到风险、权衡、失败场景、兜底方案

评分建议：

- 0-1：明显缺失核心内容
- 2-3：答到一部分，但深度或结构不足
- 4：内容较完整，有一定技术深度
- 5：内容完整、细节扎实、考虑了权衡与验证

输出要求：

- 只使用中文
- 建议必须可执行
- 不要泛泛写“继续努力”
